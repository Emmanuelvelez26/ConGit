🧩 Proyecto: Sitio web personal / portafolio simple
Un sitio con varias páginas conectadas entre sí, como si fuera tu primera web “real”.
📄 Páginas que vas a crear
1.	Inicio – Presentación
2.	Sobre mí – Info personal o ficticia
3.	Proyectos – Lista de trabajos (reales o inventados)
4.	Blog / Artículos – 2–3 posts cortos
5.	Contacto – Formulario
6.	Extras – Detalles y mejoras
________________________________________
📆 Plan de 6 días (1–2 h por día)
🟢 Día 1 – Estructura base
Objetivo: armar el esqueleto del sitio
•	Crear carpetas:
•	/proyecto
•	  /img
•	  index.html
•	  about.html
•	  projects.html
•	  blog.html
•	  contact.html
•	Usar correctamente:
o	<!DOCTYPE html>
o	<html>, <head>, <body>
o	<header>, <nav>, <main>, <footer>
•	Menú de navegación con <a> que conecte todas las páginas
✅ Al final del día: puedes navegar entre páginas sin errores
________________________________________
🟡 Día 2 – Contenido semántico
Objetivo: practicar etiquetas importantes de HTML
•	Usar:
o	<h1> a <h3>
o	<p>, <strong>, <em>
o	<section>, <article>
o	<ul>, <ol>, <li>
•	Escribir texto real (aunque sea inventado)
•	En “Sobre mí”: una mini historia personal
________________________________________
🟠 Día 3 – Imágenes y multimedia
Objetivo: trabajar con recursos visuales
•	Agregar imágenes con <img>
•	Usar:
o	alt correctamente
o	rutas relativas
•	Crear una sección de proyectos con imagen + descripción
•	(Opcional) agregar un video o iframe de YouTube
________________________________________
🔵 Día 4 – Formularios
Objetivo: dominar formularios HTML
•	En contact.html:
o	<form>
o	<label>
o	<input> (text, email, password)
o	<textarea>
o	<button>
•	Usar atributos:
o	required
o	placeholder
o	name
⚠️ No importa que no funcione “de verdad”, el foco es HTML
________________________________________
🟣 Día 5 – Tablas y detalles
Objetivo: completar lo menos usado pero importante
•	Crear una tabla:
o	Horario
o	Lista de habilidades
o	Plan de estudios
•	Usar:
o	<table>, <tr>, <th>, <td>
•	Agregar:
o	<hr>
o	<br>
o	<footer> con redes sociales
________________________________________
🔴 Día 6 – Revisión y mejoras
Objetivo: pensar como dev
•	Revisar:
o	Indentación
o	Nombres de archivos
o	Enlaces rotos
•	Validar HTML (si quieres)
•	(Opcional) agregar:
o	favicon
o	comentarios <!-- -->
o	un archivo README.txt
________________________________________

✔ Navegación
✔ Formularios
✔ Buenas prácticas

Tener en cuenta en CSS
Imágenes con tamaño fijo

width="250" y height="250" funciona, pero no es responsive.

Mejor usar CSS para controlar tamaño (max-width: 100%; height: auto;)
